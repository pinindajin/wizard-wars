# Half-Size Arena And Lava Exit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Scale the no-cliff lava arena to half its current width and height, and allow grounded players to walk from lava back onto land without jumping.

**Architecture:** Keep the arena generator as the source of truth. The generator reads the committed 4224x3392 source art, writes half-size 2112x1696 runtime/editor assets, scales prop sprites, colliders, lava/walkable rectangles, spawns, and metadata, then the existing exporter regenerates the Tiled JSON and collider constants. Movement keeps lava terrain sampling/damage, but removes the grounded lava center-containment gate so normal movement can cross lava/land boundaries while props and bounds still block.

**Tech Stack:** TypeScript, Bun, Sharp, Phaser Editor 2D scene JSON, Phaser 4, Vitest, Playwright, GitHub Actions, Dokploy.

---

## Requirements

- The arena map must be half its current size in each direction: `4224x3392` becomes `2112x1696`.
- The visible map should no longer be displayed at the oversized scale shown in the reported screenshot.
- Phaser Editor scene data must remain openable/editable with a 2112x1696 border, base image, prop images, and rectangle overlays.
- Runtime `arena-base.png`, prop sprites, generated review images, `arena.json`, layout constants, colliders, and metadata must agree on the new half-size coordinate system.
- Broadphase cells remain 64px, so `ARENA_COLS` becomes `33` and `ARENA_ROWS` becomes `27`.
- Player spawns must remain on land, inside bounds, distinct, and clear of prop colliders.
- Players must be able to walk from land into lava and from lava back onto land without jumping.
- Lava should still sample as lava, deal lava behavior through existing terrain state updates, and never create cliff/stumble behavior in this no-cliff arena.
- PR must target `main`, pass review/CI, merge to `main`, promote `main` to `prod`, wait for Dokploy deploy, then smoke test prod.

```ts
const ARENA_OUTPUT_SCALE = 0.5
const OUTPUT_REGION_CELL_PX = REGION_CELL_PX

function scaled(value: number): number {

}
```
